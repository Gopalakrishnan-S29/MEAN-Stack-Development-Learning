📦 Flexbox Mastery

Flexbox is a one-dimensional layout method for arranging items in rows or columns. It's perfect for building flexible and responsive layouts.

Key Flexbox Properties

Property Description Example Values
display: flex Enables flexbox layout flex, inline-flex
flex-direction Sets direction of flex items row, column, row-reverse, column-reverse
justify-content Aligns items along main axis flex-start, flex-end, center, space-between, space-around, space-evenly
align-items Aligns items along cross axis flex-start, flex-end, center, stretch, baseline
flex Shorthand for flex-grow, flex-shrink, flex-basis 1, 2 1 100px
gap Space between flex items 10px, 20px, 10px 20px

Example 1: Basic Flexbox Layout

Display flex items in a row with equal space:

Item 1
Item 2
Item 3
.flex-container {
    display: flex;
    gap: 10px;
    justify-content: space-between;
}

Items with different flex values grow proportionally:

📐 Box Model

The CSS Box Model describes the rectangular boxes generated for elements in the document tree. Every element consists of content, padding, border, and margin.

Box Model Layers (Inside to Outside):

  • Content: The actual content (text, images, etc.)
  • Padding: Space inside the border, around the content
  • Border: A border surrounding the padding
  • Margin: Space outside the border, between elements

Visual Example:

Content Area
(with padding & border)

Box-Sizing Property:

content-box (default): Width & height only include content

box-sizing: content-box;
width: 200px;
padding: 20px;
border: 5px solid #333;
/* Total width = 200 + 20*2 + 5*2 = 250px */

border-box: Width & height include padding and border

box-sizing: border-box;
width: 200px;
padding: 20px;
border: 5px solid #333;
/* Total width = 200px */

Best Practice:

Always use box-sizing: border-box on all elements for predictable sizing:

* {
    box-sizing: border-box;
}

Margin Collapse:

Vertical margins between block elements collapse to the larger margin value:

.element-1 { margin-bottom: 20px; }
.element-2 { margin-top: 30px; }
/* Actual gap between elements = 30px, not 50px */

📍 Positioning & Z-Index

The position property defines how an element is positioned in the document flow.

Position Types:

Position Description Use Case
static Default. Element flows in document (can't use top, right, bottom, left) Default behavior
relative Positioned relative to its normal position (stays in document flow) Slight adjustments, positioning context for children
absolute Positioned relative to nearest positioned ancestor (removed from flow) Overlays, modals, tooltips
fixed Positioned relative to viewport (removed from flow, stays on screen) Headers, footers, floating buttons
sticky Hybrid - positioned initially, then fixed when scrolled Navigation bars, table headers

Positioning Example:

Relative (offset 20px top & left)

Z-Index (Stacking Context):

Control the stacking order of overlapping elements. Higher z-index appears on top.

z-index: 1
z-index: 2
.element-1 { position: absolute; z-index: 1; }
.element-2 { position: absolute; z-index: 2; }
/* Element 2 appears on top */

📱 Responsive Design

Responsive design ensures your website looks good on all devices: desktop, tablet, and mobile.

Responsive Design Techniques:

1. Mobile-First Approach:

/* Mobile (base) */
.container { width: 100%; }

/* Tablet */
@media (min-width: 768px) {
    .container { width: 750px; }
}

/* Desktop */
@media (min-width: 1024px) {
    .container { width: 1000px; }
}

2. Flexible Layouts with Flexbox:

.flex-container {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
}

.flex-item {
    flex: 1 1 300px; /* grow, shrink, basis */
}

3. CSS Grid for Responsive Layouts:

Auto-fit automatically creates columns based on available space:

.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: 20px;
}

This creates as many 250px+ columns as fit in the container.

4. Responsive Images:

/* Make images responsive */
img {
    max-width: 100%;
    height: auto;
    display: block;
}

/* Responsive image with srcset */
<img src="small.jpg"
     srcset="medium.jpg 768px, large.jpg 1024px"
     alt="Responsive image">

5. Responsive Typography:

/* Base mobile font size */
body { font-size: 16px; }

@media (min-width: 768px) {
    body { font-size: 18px; }
}

/* Or use viewport units */
h1 { font-size: clamp(1.5rem, 5vw, 3rem); }
/* Min: 1.5rem, Preferred: 5vw, Max: 3rem */

Common Breakpoints:

Device Breakpoint Common Width
Mobile (small) < 576px 320px - 576px
Tablet 768px - 991px 768px - 991px
Desktop > 1024px 1024px+
Large Desktop > 1200px 1200px+

Viewport Meta Tag (Required for Mobile):

<meta name="viewport" 
      content="width=device-width, initial-scale=1.0">

This ensures proper scaling on mobile devices.

✅ Quick Reference Checklist

  • ✓ Use box-sizing: border-box for all elements
  • ✓ Use Flexbox for 1D layouts (rows or columns)
  • ✓ Use CSS Grid for 2D layouts
  • ✓ Mobile-first approach: start with mobile, add complexity for larger screens
  • ✓ Include viewport meta tag in HTML head
  • ✓ Use relative units (%, rem, em) instead of fixed pixels when possible
  • ✓ Test on multiple devices and use DevTools device emulation
  • ✓ Use max-width on container for desktop layouts
  • ✓ Make images responsive with max-width: 100%
  • ✓ Use media queries to adapt layouts for different screen sizes
